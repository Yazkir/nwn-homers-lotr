WHY THIS FILE EXISTS
--------------------
The 14 items currently sitting in the quarantine chest (ZEP_CR_QUARANTINE,
House of Homer) were sequestered BEFORE provenance was stamped onto the item
object, so the chest snapshot carries no submitter info. The transient
quarantine_*_info rows that DID hold it were deleted the moment a DM first
opened the chest (zep_cr_qrestore.nss). Player/character info below was instead
recovered by cross-referencing each item's ResRef against the server's logged
"[FORGE DISPUTE] ... sequestered from" / "Illicit donations item ... reclaimed
from" lines in anvil.log.

Going forward this archaeology is unnecessary: items are now stamped with a
QUARANTINE_INFO local variable that rides with the object into the chest and
survives until the item is physically removed (see forge_inc.nss /
welloferuenter.nss; bin/list-forge-quarantine.py reads it back).
