BACKGROUND
------------------------------------------------------------------------------
The area-leash anti-kiting system teleports a creature back to its spawn
area if a player leads it out (enforced in leash_to_area.nss on every area's
OnEnter). That requires each creature to have a 'spawn' LocalLocation home.

~507 creatures store 'spawn' themselves at OnSpawn via nw_c2_default9
(reached through x2_def_spawn, the module default) / nw_c2_omnivore /
nw_c2_herbivore. Associates (nw_ch_ac9, ww_ch_ac9) are intentionally NOT
leashed (they follow their PC) and are excluded from this list.

The creatures BELOW use a custom, stock, or blank OnSpawn that never stores
'spawn', so without intervention they could be kited freely.

HOW THIS WAS ADDRESSED  (final approach)
------------------------------------------------------------------------------
Home is recorded at the creature's own OnSpawn -- which always runs in the home
area, whatever the spawn mechanism (placed at area load, encounter,
MWSpawnAtWaypoint, any CreateObject). An earlier module-load sweep was scrapped
because many creatures spawn on area-enter and don't exist at module load.

Each creature below was made to store 'spawn' at OnSpawn:
  - 15 custom spawn scripts (+ nw_ch_ac9, ww_ch_ac9): the line
        SetLocalLocation(OBJECT_SELF,"spawn",GetLocation(OBJECT_SELF));
    was inserted at the top of main().
  - 8 blank-OnSpawn real monsters: ScriptSpawn set to leash_spawn (a new no-AI,
    store-only OnSpawn).
  - 41 ds_* Dungeon-Solitaire pieces: ScriptSpawn set to leash_spawn (surgically;
    bin/gen-ds-blueprints.py make_inert also updated for future regen).
  - 10 nw_c2_dropin9 / 1 nw_c2_bat9 / 2 nw_c2_dimdoors (stock, no source):
    ScriptSpawn set to thin wrappers sp_dropin9 / sp_bat9 / sp_dimdoors that
    store 'spawn' then ExecuteScript the unchanged stock script.

A build-time gate (tests/smoke-test -> tests/check_spawn_leash.py, run by
nwn-manager repack) now FAILS the build if any creature lacks both a
spawn-storing OnSpawn and the NO_LEASH=1 opt-out, so this gap can't silently
return. To opt a creature OUT (intended travelers), set local int NO_LEASH=1.
